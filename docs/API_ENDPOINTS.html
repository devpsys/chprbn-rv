Global conventions

ItemDetail
ProtocolHTTPS
JSON bodyapplication/json where applicable
verified_atEpoch milliseconds (UTC)
Naming: Login DTOs use camelCase keys for Gson. License lookup response uses snake_case matching LicenseRecordResponseDto. Verified sync request uses snake_case via @SerializedName.

Base URL

Configured in app: https://chprbn.gov.ng/api/v1/

Paths below are relative (e.g. auth/login → full URL ending with /api/v1/auth/login).

Authentication

For protected routes, use:

Authorization: Bearer <access_token>
Client note: OkHttp in the app does not yet attach this header globally. Implement server-side auth as required; the mobile client should add an interceptor when endpoints are locked down.

Standard error payload (login)

Parsed as ApiErrorDto:

{
  "message": "Invalid email or password."
}

5.4 Sync verified record

POST practitioners/verified-sync

One POST per local verification (sequential uploads).

Request body

FieldTypeRequiredDescription
license_numberstringYesRegistration number
verification_locationstringYesVerification location
practitioner_presentbooleanYesPractitioner present
remarkstringYesRemark
verified_atint64YesEpoch ms UTC
{
  "license_number": "MD-99201-B",
  "verification_location": "Lagos University Teaching Hospital — Ward 4",
  "practitioner_present": true,
  "remark": "ID matched; license card presented.",
  "verified_at": 1710858600000
}

Success

200, 201, or 204. Body may be empty.

Failure

400, 401, 409, 422, 500. Example:

{ "message": "license_number is required" }

Data models summary

  • User — id, email, fullName, permissions, userPhoto, role, staffId, unit, organization, lastLoginAt; login adds accessToken.
  • License record — snake_case JSON as in license lookup response.
  • Verified license (app) — full row in Room; API sync sends only verification fields + license_number + verified_at.
